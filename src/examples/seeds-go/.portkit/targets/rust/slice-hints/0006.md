# Rust Slice Hint — 0006 Store.Get lookup by id (EPIC-002-S3)

PRESCRIPTIVE Rust guidance for porting `Store.Get` (Go `src/examples/seeds-go/store.go:52-61`)
plus the two package facts it relies on: the `ErrNotFound` sentinel (`store.go:13`) and the
`Item` struct (`item.go:13-18`). One way only — do not deviate.

This slice ONLY adds the `get` method to the already-existing Rust `Store`. The `Store` type,
its `Mutex<StoreInner>` field, `new()`, and `create()` come from the prerequisite slice
(EPIC-002-S2) — treat them as existing. Do NOT redefine them here.

The HTTP `GET /items/{id}` call site (`server.go:46-58`) is NOT built in this slice. It is
context only; it shows that the caller distinguishes not-found via `errors.Is(err, ErrNotFound)`
(`server.go:53`), which dictates that the Rust error MUST be a single matchable variant.

---

## 1. Required crates / APIs (no alternatives)

- `std::collections::HashMap` — the store map (key `i64`, value `Item`).
- `std::sync::Mutex` — already wrapping `StoreInner` per the prerequisite slice. Lock with
  `.lock().unwrap()` (poisoning mirrors Go's no-poisoning model; see porting-hazards.md H2).
- NO new third-party crate is introduced by this slice. `Item`'s `serde`/`chrono` derives and
  the `AppError` enum already exist from earlier slices.

Do NOT use `tokio::sync::Mutex` (the critical section has no `.await`). Do NOT introduce
`thiserror`/`anyhow` for this slice.

## 3. The `NotFound` error (sentinel parity)

Go's `ErrNotFound = errors.New("item not found")` (`store.go:13`) is ONE shared package variable;
the HTTP caller matches it by identity via `errors.Is` (`server.go:53`). In Rust this is a single
enum variant on the shared `AppError` enum (established in dependency-map.md item 4 — do NOT
create a new error type for this slice):

```rust
enum AppError {
    NameRequired,
    NameTooLong,
    NotFound,   // <-- the variant this slice returns; == Go ErrNotFound
    Duplicate,
}
```

- `get` returns `AppError::NotFound` on a missing key — the SAME variant the caller matches on
  with `matches!(err, AppError::NotFound)` (the Rust analog of `errors.Is(err, ErrNotFound)`).
- `AppError::NotFound`'s `Display` MUST emit EXACTLY `item not found` (grounded `store.go:13`).
  The HTTP layer surfaces this text as the 404 body (`server.go:54`), so it is observable.
  Message text is NOT asserted by any Go test (`store_test.go:74-89` checks identity only) but is
  SOURCE-GROUNDED — implement it verbatim.

## 4. The `get` method — EXACT implementation

Signature (the Rust port of `func (s *Store) Get(id int) (Item, error)`, `store.go:53`):

```rust
impl Store {
    /// Returns the item with the given id, or AppError::NotFound. (store.go:52-61)
    pub fn get(&self, id: i64) -> Result<Item, AppError> {
        let g = self.inner.lock().unwrap();          // store.go:54  s.mu.Lock()
        match g.items.get(&id) {                       // store.go:56  it, ok := s.items[id]
            Some(it) => Ok(it.clone()),                // store.go:60  return it, nil
            None => Err(AppError::NotFound),           // store.go:57-58 return Item{}, ErrNotFound
        }                                              // guard `g` drops here = store.go:55 defer Unlock
    }
}
```

Mandatory specifics:

- Parameter type is `i64` (NOT `i32`/`usize`) to mirror Go `int` width (porting-hazards.md H8;
  Go `int` is 64-bit on these targets, `item.go:14`, `store.go:22`).
- Return type is `Result<Item, AppError>`. There is no zero-value `Item` on the error path:
  Go returns `Item{}` (`store.go:58`) but the caller never reads it on error (`server.go:53-57`),
  so `Result`'s `Err` is the exact, stricter equivalent (porting-hazards.md H15). Do NOT return a
  sentinel/default `Item` on not-found.
- `&self` (shared borrow) — `get` is a pure read; it MUST NOT take `&mut self` and MUST NOT
  mutate `items` or `next_id` (Go `Get` is a pure read, `store.go:53-61`).
- Take the lock as the FIRST statement and hold it for the whole method (mirrors `s.mu.Lock()` +
  `defer s.mu.Unlock()`, `store.go:54-55`). The guard `g` releases at end of scope = the `defer`.
  Take the lock exactly ONCE; no nested/second lock (it is not re-entrant).
- Use `HashMap::get(&id)` for an exact single-key fetch (mirrors the Go two-value map index
  `it, ok := s.items[id]`, `store.go:56`). This is exact-match on the integer key only — no
  fuzzy/prefix/name matching (porting-hazards.md, exactness note; `store.go:56`).
- `.clone()` the `Item` out of the map WHILE holding the guard, then return the owned clone
  (porting-hazards.md H1). Do NOT return a reference into the locked map.
- `g` is an immutable guard (`let g`, not `let mut g`) — `get` reads only.

## 5. Behavior table (exactly two outcomes — no others, no panics)

| Condition (Go) | Go return (`store.go`) | Rust return |
|---|---|---|
| `id` present in map | `it, nil` (l.60) | `Ok(item.clone())` |
| `id` absent | `Item{}, ErrNotFound` (l.57-58) | `Err(AppError::NotFound)` |

- Any `i64` is a valid input. No range validation (Go validates none; `store.go:53-61`).
- Stored keys are positive ints assigned from 1 by `create` (`store.go:28` `next_id:1`,
  `store.go:46-48`). Therefore `get(0)` and `get(<negative>)` NEVER match → `Err(NotFound)`.
- `get` NEVER panics on the request path and returns NO error value other than
  `AppError::NotFound` (porting-hazards.md H19). The only `.unwrap()` permitted is on
  `inner.lock()` (H2/H19).

## 6. Tests for THIS slice (mirror `TestStoreGet`, `store_test.go:74-89`)

Put these in `#[cfg(test)] mod tests` in the same module as `Store` (Rust has no subtests; use
plain `#[test]` fns / loops). These are SYNCHRONOUS tests — `get`/`create`/`new` are not async,
so use `#[test]`, NOT `#[tokio::test]`.

```rust
#[test]
fn get_found_returns_item_no_error() {            // AT-1; store_test.go:76-84
    let s = Store::new();
    let created = s.create("findme").unwrap();
    let got = s.get(created.id).expect("get existing");
    assert_eq!(got.name, "findme");
    assert_eq!(got.id, created.id);
}

#[test]
fn get_missing_returns_not_found() {              // AT-2; store_test.go:86-88
    let s = Store::new();
    let err = s.get(999).unwrap_err();
    assert!(matches!(err, AppError::NotFound));   // == errors.Is(err, ErrNotFound)
}

#[test]
fn get_missing_is_err_not_some_default() {        // AT-3; store.go:58 (Result Err, no zero Item)
    let s = Store::new();
    assert!(s.get(999).is_err());                 // Rust has no zero-value Item; Err carries none
}

#[test]
fn get_exact_key_match_only() {                   // AT-4; store.go:56; ids start at 1 (store.go:28,46-48)
    let s = Store::new();
    let first = s.create("a").unwrap();           // first.id == 1
    assert_eq!(first.id, 1);
    assert!(matches!(s.get(0).unwrap_err(), AppError::NotFound));
    assert!(matches!(s.get(2).unwrap_err(), AppError::NotFound));
    let got = s.get(1).expect("get id 1");
    assert_eq!(got.id, 1);
}

#[test]
fn not_found_message_text() {                     // AT-5; store.go:13 (guard; not asserted upstream)
    assert_eq!(AppError::NotFound.to_string(), "item not found");
}
```

Notes on the tests:
- AT-3 differs from the Go phrasing on purpose: Go's "zero-value `Item`" return
  (`store_test.go` does not even assert this; doc AT-3 is grounded at `store.go:58`) has no Rust
  equivalent because `Err` carries no `Item`. Asserting `.is_err()` is the faithful port.
- `create("a").id == 1` is grounded at `store.go:28,46-48` and asserted explicitly to lock the
  off-by-one expectations of AT-4.

## 8. Pitfalls (do NOT do these)

- Do NOT return `Option<Item>` from `get` — the caller distinguishes not-found via a typed error
  (`AppError::NotFound`) to mirror `errors.Is(err, ErrNotFound)` (`server.go:53`). Use
  `Result<Item, AppError>`.
- Do NOT create a fresh per-call error value for not-found — it must be the single matchable
  `AppError::NotFound` variant (identity parity with the Go sentinel, `store.go:13`).
- Do NOT take `&mut self`, do NOT mutate any field — `get` is a pure read (`store.go:53-61`).
- Do NOT use `i32`/`usize` for the id — use `i64` (H8).
- Do NOT return a reference into the `HashMap`/guard — `.clone()` out while locked (H1).
- Do NOT use `tokio::sync::Mutex` or hold the guard across any `.await` — there is none (H2).
- Do NOT add range/name/fuzzy matching — exact integer-key lookup only (`store.go:56`).
