## 4. The `Item` type and its JSON body shape (the contract)

`store.get` returns the `Item` type. Its serialized form IS the 200 body. Define `Item` per
`porting-hazards.md:113-122`, grounded in `item.go:13-18`:

```rust
use chrono::{DateTime, Utc};
use serde::{Serialize, Deserialize};

#[derive(Clone, Serialize, Deserialize)]
struct Item {
    id: i64,                          // item.go:14  json:"id"
    name: String,                     // item.go:15  json:"name"
    done: bool,                       // item.go:16  json:"done"
    #[serde(rename = "createdAt")]    // item.go:17  json:"createdAt"  (camelCase — MUST rename)
    created_at: DateTime<Utc>,
}
```

JSON body produced (keys in this declaration order — serde emits struct fields in order, same
as Go's `encoding/json`; slice doc §2):

| JSON key | Rust type | Encoding |
|----------|-----------|----------|
| `id` | `i64` | bare integer, e.g. `1` (`item.go:14`) |
| `name` | `String` | JSON string (`item.go:15`) |
| `done` | `bool` | `true`/`false` (`item.go:16`) |
| `createdAt` | `DateTime<Utc>` | RFC 3339 string (`item.go:17`) |

Critical naming rule (`porting-hazards.md:99-126`): `id`, `name`, `done` already match
lowercase; ONLY `created_at` needs `#[serde(rename = "createdAt")]`. Do NOT use
`#[serde(rename_all = "camelCase")]`. The byte-for-byte key set is the contract.

## 5. Two behavioral divergences from Go — both test-invisible, but document them

### 5a. Trailing newline `[UNVERIFIED — known divergence]`

Go's `json.NewEncoder(w).Encode(v)` appends a single `\n`, so the Go body ends `}\n` (slice doc
§2, `server.go:79`). `axum::Json` / `serde_json::to_vec` do NOT append a trailing newline — the
Rust body ends `}` with no newline. This divergence is NOT covered by any test: the existing
test asserts only status 200 (`server_test.go:79-82`), and AT-2 below decodes the body with
`serde_json::from_slice`, which is newline-agnostic. Do NOT hand-append `\n` to chase parity —
no test requires it and it is not idiomatic. Flagged so the downstream model does not assume
the bodies are byte-identical.

### 5b. Timestamp format `[known divergence, HUMAN-DECISION]`

Go marshals `time.Time` as local-offset RFC 3339 (e.g. `-07:00`); `chrono::DateTime<Utc>`
serializes as `Z` (`porting-hazards.md:130-148`). NO test asserts the timestamp value
(`porting-hazards.md:141-144`), so this is test-invisible. Use `DateTime<Utc>`; do NOT attempt
local-offset parity unless a human requires it.
