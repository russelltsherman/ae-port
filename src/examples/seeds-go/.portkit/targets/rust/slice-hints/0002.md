## 0. Scope of THIS slice

This slice implements ONE method: `Store::create`. It does NOT implement `get`/`list`/`delete`
(those are other slices), the HTTP layer, or `ValidateName` (slice EPIC-001-S3). It DEPENDS on
kernel/prerequisite symbols listed in §1 already existing.

Reference Go body to match exactly (`store.go:33-50`):

```go
func (s *Store) Create(name string) (Item, error) {
    if err := ValidateName(name); err != nil {
        return Item{}, err
    }
    trimmed := strings.TrimSpace(name)

    s.mu.Lock()
    defer s.mu.Unlock()
    for _, it := range s.items {
        if strings.EqualFold(it.Name, trimmed) {
            return Item{}, ErrDuplicate
        }
    }
    it := Item{ID: s.nextID, Name: trimmed, CreatedAt: s.now()}
    s.items[it.ID] = it
    s.nextID++
    return it, nil
}
```

Decisions (do not deviate):
- Take `&self` (NOT `&mut self`). Interior mutability lives in `Mutex<StoreInner>`

- Return an OWNED `Item` (a clone), never a borrow into the locked map (`porting-hazards.md` H1).

## 3. Exact body — step by step (preserve this logical order; it is observable)

Ordering is mandated by `store.go:34-49` and `porting-hazards.md` H10: validate → trim → lock →
duplicate-scan → construct/insert/increment → return.

```rust
pub fn create(&self, name: &str) -> Result<Item, AppError> {
    // 1. Validate FIRST, before locking. Propagate the error UNCHANGED.
    //    (store.go:34-36 — returns err verbatim; mutex never taken on validation failure.)
    validate_name(name)?;

    // 2. Trim. The TRIMMED value is what is stored AND compared. (store.go:37)
    let trimmed = name.trim();

    // 3. Lock for the remainder. RAII guard == Go's `defer s.mu.Unlock()`. (store.go:39-40)
    //    .unwrap() on lock is the accepted choice for this port (porting-hazards H2; Go has
    //    no poisoning concept).
    let mut g = self.inner.lock().unwrap();

    // 4. Case-insensitive duplicate scan over ALL existing items. (store.go:41-44)
    //    Use eq_ignore_ascii_case — the agreed default for EqualFold (porting-hazards H13).
    for it in g.items.values() {
        if it.name.eq_ignore_ascii_case(trimmed) {
            return Err(AppError::Duplicate);
        }
    }

    // 5. Construct. id = current next_id; name = trimmed; created_at = now(); done defaults false.
    //    (store.go:46 — Done is never set, so it is the zero value `false`.)
    let it = Item {
        id: g.next_id,
        name: trimmed.to_string(),
        done: false,
        created_at: (self.now)(),
    };

    // 6. Insert keyed by the new id, THEN increment next_id. Increment ONLY on success.
    //    (store.go:47-48)
    g.items.insert(it.id, it.clone());
    g.next_id += 1;

    // 7. Return the inserted item (owned clone) with no error. (store.go:49)
    Ok(it)
    // guard `g` drops here, releasing the lock (== defer Unlock).
}
```

### Mandatory specifics (a weaker model must get these exactly right)

- **`validate_name(name)?` comes before everything.** On its `Err`, return immediately; the lock
  is NEVER acquired, no scan runs, no insert happens, `next_id` is NOT incremented
  (`store.go:34-36`, `store.go:45` ordering; spec §2 item 1).
- **Propagate the validation error unchanged.** `?` returns the same `AppError` variant
  (`NameRequired` or `NameTooLong`). Do NOT wrap, map, or re-message it (`store.go:35`).
- **Trim with `name.trim()`** (Go `strings.TrimSpace`, `store.go:37`; `porting-hazards.md` H14).
  Use the trimmed `&str` both for the dup-scan comparison and (via `.to_string()`) for the stored
  name. Storing the untrimmed name is a BUG.
- **Duplicate comparison uses `eq_ignore_ascii_case`** against each stored `it.name`
  (`store.go:42`; `porting-hazards.md` H13). The candidate is already trimmed, so detection is
  both case-insensitive AND trim-insensitive — matching `store_test.go:65,69`. Do NOT use
  `==`, `.to_lowercase() == .to_lowercase()`, or `.contains()`.
- **First match wins:** return `Err(AppError::Duplicate)` on the first matching item; do not
  continue scanning (`store.go:42-43`).
- **`done: false`** is set explicitly because Rust has no implicit zero value; Go leaves it zero
  (`store.go:46` never assigns `Done`; spec §1 item 7, §2 item 5).
- **`created_at: (self.now)()`** — call the injected clock closure, NOT `Utc::now()` directly, so
  tests can inject a fixed time (`store.go:46` calls `s.now()`; `dependency-map.md` item 9).
- **Insert THEN increment.** `g.items.insert(it.id, it.clone()); g.next_id += 1;`
  (`store.go:47-48`). The clone is required because you also return `it` (H1). Incrementing
  before insert, or incrementing on a rejected create, is a BUG — `next_id` advances ONLY on a
  successful insert (spec §2 "Ordering guarantees"; `store.go:46-48`).
- **IDs are sequential from 1, +1 per success.** Guaranteed by `next_id` starting at 1
  (`store.go:28`) and `+= 1` only here. A rejected create (validation or duplicate) consumes no
  ID; the next success reuses the would-be ID (spec §2 ordering item 3; `store.go:35,43,46-48`).
- **Return the owned `it` clone**, never a reference into `g.items` (H1). The guard `g` must drop
  before/at return; returning `Ok(it)` where `it` is owned does this automatically.

## 5. Tests for THIS slice (port of `store_test.go:31-72`)

Put under `#[cfg(test)] mod tests`. Plain `#[test]` (no async needed — `create` is synchronous;
`dependency-map.md` "Test-framework dependencies"). Match these exact assertions:

1. **Sequential IDs from 1.** Fresh store: `create("  Write tests  ")` → `id == 1`; then
   `create("Second")` → `id == 2`. (`store_test.go:38,50`; `store.go:28,46-48`.)
2. **Stored name trimmed.** `create("  Write tests  ")` → returned `name == "Write tests"`.
   (`store_test.go:34,41-42`; `store.go:37,46`.)
3. **Empty name rejected.** `create("")` → `Err(AppError::NameRequired)`; nothing stored.
   (`store_test.go:58`; `store.go:34-36`.) Assert via `matches!(err, AppError::NameRequired)`.
4. **Over-length rejected.** `create(&"y".repeat(101))` (101 chars, `MaxNameLen + 1`) →
   `Err(AppError::NameTooLong)`; nothing stored. (`store_test.go:61`; `store.go:34-36`,
   `item.go:33`.) Use `.repeat(101)` (Go `strings.Repeat`, `dependency-map.md` item 5).
5. **Duplicate is case- AND trim-insensitive.** After `create("Unique")` succeeds,
   `create("  unique ")` → `Err(AppError::Duplicate)`; nothing additional stored.
   (`store_test.go:65,69`; `store.go:41-44`.)
6. **Success returns populated item + Ok.** A successful `create` returns `Ok(item)` with the
   assigned id and trimmed name. (`store_test.go:34-43`; `store.go:46-49`.)

Notes:
- Go uses `errors.Is(err, ErrXxx)` (`store_test.go:58,61,69`); the Rust analog is
  `matches!(err, AppError::Xxx)` (`dependency-map.md` item 4).
- The exact duplicate message string `"an item with that name already exists"`
  (`store.go:14`; spec §4 item 5) is carried by `AppError::Duplicate`'s `Display` impl (kernel
  responsibility). This slice does not need to assert the string unless the kernel test does.
- Concurrency is NOT tested in the source (`store_test.go` has no concurrent test); do not add
  one. [UNVERIFIED by test] thread-safety is a contract requirement only (spec §2 invariants).

## 6. Anti-patterns — do NOT do these

- Do NOT use `&mut self` and a bare `HashMap` field; state MUST live inside `Mutex<StoreInner>`
  (H2). axum shares `Arc<Store>`, which cannot give `&mut`.
- Do NOT use `tokio::sync::Mutex` — the critical section has no `.await` (H2).
- Do NOT use `name.len()` anywhere in length logic — that is bytes; length is code points via
  `.chars().count()` (this lives in `validate_name`, but never reimplement it with `.len()`, H12).
- Do NOT trim only for comparison while storing the raw name, or vice versa — both use `trimmed`
  (`store.go:37,42,46`).
- Do NOT increment `next_id` on the validation-error or duplicate-error path (`store.go:35,43`).
- Do NOT return a reference into the locked map; clone out (H1).
- Do NOT use `.to_lowercase()` for the duplicate fold and claim Go parity — `eq_ignore_ascii_case`
  is the chosen default (H13). [Full Unicode `EqualFold` parity is HUMAN-DECISION-REQUIRED per
  `dependency-map.md`; all existing tests are ASCII so the default passes them.]
- Do NOT call `Utc::now()` directly in `create`; call the injected `self.now` closure
  (`store.go:46`).
